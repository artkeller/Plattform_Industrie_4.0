# Use Case description
**Use Case: Generation of ABAC Rules for Access to an Industry 4.0 Data/Collaboration Space**
The scenario describes a company within the EU that manufactures robotics machines and has suppliers from various countries, while also providing access to a shared development platform with engineers and software developers from additional countries. In accordance with its own policies, the company must regularly monitor changes in import/export regulations, data privacy regulations, regulations for data exchange, and regulations for data access in order to perform appropriate risk assessments and decide whether access rules need to be changed. These changes to the access rules are ultimately to be provided as executable code, as in Variant 1.

An additional possible result was the question if the same source data that can aggregated in Stage 3 (see Figure 1) to be used as a basis for a report to management and the legal department. Corresponding risk assessments could then be performed with this data (e.g., for new import/export regulations, data exchange regulations...).
