# Use Case description
**Generation of ABAC Rules for Access to an SAP System**

The objective of this variant is to generate finally executable code for attribute-based access control from a query to an LLM for assessing the security relevance/criticality of one or more SAP objects (e.g., so-called transactions, reports, function modules, services...) that are located in an SAP S4/H system, which can be executed in a ZTA. The ZTA machine can be implemented arbitrarily and can, for example, execute XACML (an XML-based open source policy language), C++, or Python code.



The SAP use case was selected as a preparatory use case because domain-specific knowledge was available for evaluating the answers.
